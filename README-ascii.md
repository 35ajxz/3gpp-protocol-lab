# 3GPP Protocol Lab

> 面向 TS 38.331 Rel-17 的 3GPP RRC 协议安全研究工程
> 目标：把真实规范转化为可信协议模型，并基于协议状态机进行逻辑漏洞挖掘与有状态模糊测试。

常见问题与补充说明请参考 `FAQ/` 目录。

---

## 一、项目定位

这不是一个"能读懂规范"的项目，而是一个**把规范打通成可执行、可复现测试资产**的工程。

核心研究链路：

```
真实 3GPP 归档 → 正文提取 → 过程切片 → 结构化状态机(EFSM)
→ 模型校验 → 逻辑漏洞测试 → 协议二进制/抓包/回放 → 目标系统注入
```

当前覆盖规范过程：

| 章节 | 过程 |
|---|---|
| 5.3.3 | RRC Connection Establishment（连接建立） |
| 5.3.5.11 | Full Configuration（完整配置） |
| 5.3.13 | RRC Connection Resume（连接恢复） |

## 九、设计原则

```
  ┌─────────────────────────────────────────────────────────────┐
  │  1. 真实资料优先     不用玩具语料，直接以真实规范归档为起点     │
  │  2. LLM 只做提取    文档导入/约束校验/逻辑判定/落盘全部本地代码 │
  │  3. 漏洞必须可复核   必须能回到规则、轨迹和产物，而非自然语言    │
  │  4. 输出必须可消费   状态机不是终点，它必须能喂入下一层测试      │
  └─────────────────────────────────────────────────────────────┘
```
